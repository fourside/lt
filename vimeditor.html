他のエディタとの違いは?

  • モードを切り替えながら編集するよ
  • キーバインドが変

vimのモード

  • 主に使うのはふたつだけ
    • normal mode ... 編集
    • insert mode ... 文字の入力
    • オマケ command mode ... コマンドの実行

モードの恩恵

  • 文字入力をモードとして切り分けることで、修飾キーを使わずにキーを組み合わせることができる
  • エディタの機能がキーボードでまかなえる!

mode間の行き来

  • a, iでinsert modeに。
  • エスケープキーでnormal modeに。

エスケープキーは遠い！

  • Ctrl-[ を押そう
  • 他の遠いキーも別のキーがあるよ(vimとは関係ないけど)
    • Enter => Ctrl-M
    • BS => Ctrl-H

変なキーバインド

  • カーソル移動がjkhl
    • h j k l
    • ←↓↑→
  • とっつきにくい
  • 遠く離れたカーソルキーを押さなくていい！

慣れるには

  • vimtutorコマンド
    • vimをインストールするとついてくるコマンド
    • テキストの指示に従って、そのテキストを編集していく
    • windows(kaoriya版)なら、:Tutorial

ちょっと便利なこと

  • gf
  • テキストオブジェクト

gfコマンド

  • カーソル位置にファイル名があるとき、そのファイルが開ける
  • 他のファイルをインクルードしてるとき便利
    • httpd.conf
    • :h gf

テキストオブジェクト

  • ある範囲の文字列をオブジェクトとみなして、そいつに対する操作をする
    • puts "hello, #{you^rname}"
    • ^がカーソル位置だとして、di{と打つと...
    • puts "hello, #{^}"

「di{」

  • dで、デリート
  • iで、テキストオブジェクトの中(inner)
  • {で、テキストオブジェクトの指定
  • 「di{」は、「delete inner {.」
  • :h text-objects

補完

    • バッファからの補完

    • 便利なプラグインがある

自動コンパイル

  • 保存時にコンパイルしてくれる機能
    • スクリプト系だとシンタックスチェック
    • エラー行をリスト表示してくれる

    • :h make

関数ジャンプ

  • 関数の定義元にジャンプする
    • ctagsでタグファイルを作って、場所をvimに教える
    • :h tags
    • taglist.vimプラグインが便利

エディタからの実行

  • 今開いているバッファの中身を実行したい
    • :!ruby %
    • quickrun.vimというプラグインが便利
      • ファイルタイプから実行するコマンドを決定してくれる
